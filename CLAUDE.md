# Blue Cap IT Main Website (bluecapit.com)

## About This Project

This is the marketing website for Blue Cap IT, an IT services business and official Verkada Gold Partner. The site showcases security solutions including cameras, access control, sensors, alarms, intercoms, and command center software.

**The owner is not a developer and uses Claude Code to manage the site through natural language.** Always explain what you're about to change and why in plain English before making any edits. Do not use jargon without explanation.

## Tech Stack

- **Astro** — static site generator (builds plain HTML/CSS, no JavaScript framework)
- **Fonts:** Google Fonts (Sora for headings, DM Sans for body text)
- **Styling:** Single `src/styles/style.css` imported in the layout
- **Analytics:** Google Analytics (gtag.js, ID: G-HN39P8Z2HR)
- **Hosting/Deployment:** Static files (the `dist/` folder is the deployable output)

## Common Commands

- `npm run dev` — Start local dev server at http://localhost:4321
- `npm run build` — Build the site into `dist/`
- `npm run preview` — Preview the built site locally

## File Structure

```
/ (root)
  astro.config.mjs        — Astro configuration (outputs .html file extensions)
  package.json             — Project dependencies (just Astro)
  tsconfig.json            — TypeScript config (required by Astro)

  src/
    layouts/
      MainLayout.astro     — Shared layout: head, nav, mobile menu, footer, hamburger script
    pages/
      index.astro          — Homepage
      about.astro          — About page
      services.astro       — Services overview
      cameras.astro        — Verkada cameras
      access-control.astro — Access control systems
      alarms.astro         — Alarm systems
      sensors.astro        — Environmental sensors
      intercom.astro       — Intercom systems
      command.astro        — Command center software
      contact.astro        — Contact page
    styles/
      style.css            — All site styles

  public/
    logo.png               — Blue Cap IT logo
    *.png                  — Product/marketing images (Verkada product renders)

  dist/                    — Built site (generated by `npm run build`, not committed)
```

## Key Conventions

- All pages use `MainLayout.astro` — nav, footer, analytics, and CSS are defined once there
- Each page passes its own `title` and optional `description` to the layout
- To change the nav or footer, edit only `src/layouts/MainLayout.astro`
- Images in `public/` are served at the root URL (e.g., `public/logo.png` → `/logo.png`)
- Logo and partner badge images are embedded as base64 data URIs in the layout (nav and footer)
- The Astro build outputs `.html` file extensions to preserve the original URL structure

## Rules for Making Changes

1. **Explain first.** Before editing any file, describe what will change and why in simple terms.
2. **Keep it simple.** This is a small business marketing site — avoid over-engineering.
3. **Nav/footer changes go in the layout.** Edit `src/layouts/MainLayout.astro` — it applies to all pages automatically.
4. **Respect the design.** The site uses Sora for headings and DM Sans for body text. Maintain this typography.
5. **Images go in `public/`.** They'll be available at the root URL.
6. **No new dependencies** unless explicitly approved — the site should remain simple to deploy.
7. **Do not push to the remote** without explicit approval.
8. **Run `npm run build`** after changes to verify the site builds without errors.
